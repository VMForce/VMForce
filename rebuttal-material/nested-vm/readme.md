# Virtual Machines in Themida
Themida has 6 virtual machines, namely tiger, fish, dolphin, puma, eagle, and shark. Each virtual machine has three color configurations, namely white, red, and black, and the complexity of the virtual machine increases in sequence as the configuration changes.

In fact, among the six virtual machines, only tiger, fish, and dolphin are basic non-nested virtual machines, while the remaining three virtual machines are generated by nesting the three basic virtual machines. In particular, eagle is generated by nesting dolphin within fish, puma is generated by nesting fish within tiger, and puma is generated by nesting tiger within fish.

# Experiment
We selected sample rq2-4 and protected it with shark-white VM. As the file "trace.txt" shows, 12052 virtualized conditional transfers are recognized by VMForce. By force-execution, we found that only 5797 and 5798 were in inner layers and only by reversing them could we monitor the hidden malicious behaviors. It took us about 9 hours to complete the force-execution. Due to the excessive number of cycles in force-execution, the total API log file generated was extremely huge, so we only stored the key API records in file "result.txt". The results show that VMForce found 28 hidden syscalls in consistent with PMP.
